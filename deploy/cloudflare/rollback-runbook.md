# Cloudflare 回滚说明

## 目标

在不触碰后端密钥和不扩大公网副作用的前提下，快速把受影响入口从异常版本切回可控状态。

## 一级回滚

适用：上游服务异常、局部路由错误、临时需要阻断公网流量。

1. 由 human 审批后，将 `CF_MAINTENANCE_MODE` 切为开启状态。
2. 如需统一落到维护页，设置 `CF_MAINTENANCE_TARGET` 指向维护站点。
3. 验证：
   - `GET /__edge/health` 返回维护模式已开启
   - 业务入口已按预期进入维护页或 `503`

## 二级回滚

适用：Worker 脚本本身存在错误，或一级回滚无法止损。

1. 由 trusted runner 回退到上一版 Worker 发布。
2. 恢复上一个已验证的 host -> origin 路由。
3. 验证：
   - `console.*`
   - `api.*`
   - `preview.*`
   三类域名均可回到上一版行为。

## 三级回滚

适用：边缘层策略、域名或 WAF 变更引发系统性故障。

1. 撤销最近一次域名切流或回源策略变更。
2. 恢复上一份已审批的 Cloudflare 配置快照。
3. 对外入口恢复后，再排查具体故障版本。

## 回滚后必须做

1. 记录触发时间、影响域名、回滚级别与恢复时间。
2. 更新 incident 时间线。
3. 把真实回滚结果回填到后续观测与 release guard 证据中。
